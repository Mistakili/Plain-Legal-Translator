Project: PlainLegal iOS / Premium Version

This is a fork of PlainLegal — an AI-powered legal document translator that breaks down contracts into plain English with risk flags. The original was built for the DigitalOcean Gradient AI Hackathon. This fork is being turned into a production iOS app with premium features.

Current Stack:

Frontend: React + TypeScript + Vite + shadcn/ui + Tailwind + Framer Motion
Backend: Express + TypeScript + PostgreSQL (Drizzle ORM)
AI: DigitalOcean Gradient AI (OpenAI-compatible API at https://inference.do-ai.run/v1/, model llama3.3-70b-instruct, key stored as DO_GRADIENT_API_KEY)
Sessions: express-session with connect-pg-simple (PostgreSQL-backed), trust proxy enabled
Routing: wouter (frontend), Express (backend)
What already works:

Paste text or upload files (multi-file, up to 10 files, PDF/TXT/DOC/DOCX, 10MB each)
AI analysis pipeline: plain English translation, risk flags with severity levels, key terms, document type detection
AI-powered Q&A chat per document (Ask AI tab on analysis page, 20-message history, 6000-char doc truncation)
Sample documents (Freelancer Agreement, Residential Lease, SaaS ToS) for instant demo
Session-based document isolation (each browser session only sees its own documents)
Dark mode with localStorage persistence
pdf-parse v2 class-based API (new PDFParse({ data: buffer }) then parser.getText())
Production build: esbuild bundles to dist/index.cjs, pdf-parse is forced external in script/build.ts
Key files:

shared/schema.ts — Drizzle schema (documents with sessionId, chatMessages), Zod schemas (analysisSchema, riskFlagSchema)
server/routes.ts — All API routes, AI prompts (SYSTEM_PROMPT, QA_SYSTEM_PROMPT), pdf-parse lazy import via getPDFParseClass()
server/storage.ts — DatabaseStorage class, all CRUD operations scoped by sessionId
server/index.ts — Express setup, session config with PG store, trust proxy
client/src/pages/home.tsx — Upload tab (default), Paste Text tab, Try a Sample tab, document list
client/src/pages/analysis.tsx — Document analysis view with Ask AI chat
client/src/components/theme-provider.tsx — Dark mode toggle
client/src/lib/sample-documents.ts — 3 built-in sample legal documents
script/build.ts — esbuild config, pdf-parse in forceExternal list
Technical gotchas to remember:

pdf-parse v2 is a class, NOT a function: new PDFParse({ data: buffer }) → parser.getText() → result.text
pdf-parse must be lazy-imported via dynamic import() and forced external in build config
OpenAI client must be lazy-initialized via getOpenAI() factory to avoid startup crashes if API key is missing
Session cookies need trust proxy + secure: true + sameSite: "lax" to work behind Replit's reverse proxy
The connect-pg-simple store uses createTableIfMissing: true to auto-create the session table
What needs to be built for iOS/Premium:

User accounts — Replace anonymous sessions with real auth (email/password or OAuth). Users need accounts for cross-device access and subscription management.
Rate limiting — Free tier: ~3 analyses/month. Use express-rate-limit or a DB counter.
Premium subscription — RevenueCat integration (see attached knowledge base). Gate features: unlimited analyses, AI chat, multi-file upload, export.
Capacitor iOS setup — Web app served remotely via Capacitor. See attached knowledge base for config template, WKAppBoundDomains, CORS for capacitor://localhost, cookie policy (sameSite: "none" for WKWebView).
App Store compliance — Privacy policy page (/privacy), Terms of Service page (/terms), Restore Purchases button, in-app account deletion, AI consent disclosure.
Push notifications (optional) — APNs via node-apn-http2 for document analysis completion alerts.
Attached knowledge base: The file attached_assets/VEMINDER_KNOWLEDGE_BASE_*.md contains battle-tested patterns for Capacitor iOS setup, APNs push notifications, RevenueCat subscriptions, App Store compliance, and critical iOS gotchas. Read this before starting any iOS work — it will save significant debugging time.

Design system: Inter font, shadcn/ui components, Tailwind, framer-motion animations. Never nest Cards. Never use custom h-/w- on size="icon" buttons. Use AlertDialog for destructive actions.
